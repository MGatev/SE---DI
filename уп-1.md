## 1. Принципи на структурното програмиране
В програмирането съществуват различни парадигми за изграждане на програми. Програмирането може да бъде разделено на две основни парадигми: 
- Императивно - описва как се изпълнява дадена задача, чрез поредица от инструкции
- Декларативно - Oписва какво трябва да се постигне, без да се посочват конкретните стъпки за изпълнение

### Процедурното програмиране (определение)
Процедурното програмиране е императивна парадигма, при която програмата се изгражда от последователно изпълнявани инструкции, организирани във функции (наричани още процедури). Основният подход е линеен (отгоре надолу), като данните и функциите се третират като отделни компоненти. Всяка функция изпълнява конкретна подзадача и може да бъде извиквана многократно.

### Принципи на структурното програмиране
Структурното програмиране е разширение на процедурното, при който програмата се изгражда чрез ясно дефинирани управляващи конструкции и логически обособени блокове от инструкции. Основната му цел е да направи програмния код по-разбираем, по-лесен за тестване, поддръжка и разширяване.
Основен принцип на структурното програмиране е избягването на хаотични безусловни преходи, например прекомерна употреба на оператора goto. Вместо това управлението на изпълнението се реализира чрез три основни конструкции:
1.	Последователност — инструкциите се изпълняват една след друга в реда, в който са записани. 
2.	Разклонение — в зависимост от дадено условие се избира кой блок от инструкции да бъде изпълнен. Това се реализира чрез конструкции като if, if-else и switch. 
3.	Повторение — даден блок от инструкции се изпълнява многократно, докато е изпълнено определено условие. Това се реализира чрез цикли като for, while и do-while. 

Важен принцип е всяка конструкция или блок от код да има ясно определени вход и изход. Това улеснява проследяването на потока на изпълнение и намалява вероятността от логически грешки.

Структурното програмиране използва и принципа на декомпозицията — сложната задача се разделя на по-малки и по-лесно управляеми подзадачи. В процедурното програмиране тези подзадачи обикновено се реализират чрез функции и процедури. Така се постига модулност, повторна употреба на код и по-добра организация на програмата.

## 2. Управление на изчислителния процес
Управлението на изчислителния процес представлява определяне на реда, по който се изпълняват инструкциите в една програма. В процедурното програмиране програмата се разглежда като последователност от действия, които обработват данни и постепенно променят състоянието на изчислението. Програмната структура се състои от ясно дефинирани елементи, които изграждат логиката и потока на изпълнение на програмата.

### Инструкция (statement)
Инструкцията е най-елементарната единица за изпълнение в една програма. 
Това е команда, която казва на компютъра да извърши определено действие.
В езика C++ всяка инструкция завършва с точка и запетая `;`.

Някои основни типове инструкции:
- Присвояване: `a = 5;`
- Извикване на функция: `print();`
- Въвеждане/извеждане: `cin >> x; cout << x;`
- Условна инструкция: `if (x > 0) cout << "положително";`

### Израз (expression)
Изразът представлява комбинация от:
- постоянни стойности (константи) - напр. `3.14`, `'A'`, `true`;
- променливи – напр. `a`, `n`, `sum`;
- оператори – напр. `+`, `-`, `*`, `=`, `==` и др.

Изразът винаги има стойност, която се изчислява в момента на изпълнение.
Примери за изрази:
```c++
a + b;
x > y;
count++;
n % 2 == 0;
```
Изразът може да бъде част от инструкция, напр.:
```c++
int z = x + y * 2;
```

### Блок от инструкции (compound statement)
Блокът от инструкции (или съставна инструкция) позволява да се групират няколко отделни инструкции и да се третират като една логическа единица.
Това се постига чрез заграждане на инструкциите с фигурни скоби `{}`.

Блоковете се използват най-често:
- в тялото на условни конструкции (if, else, switch);
- в цикли (for, while);
- във функции – тялото на функцията е блок.
```
{
    <инструкция1>;
    <инструкция2>;
    ...
}
```

### Коментари
Коментарите са части от кода, предназначени за обяснение, пояснение или временно деактивиране на код.
Те не се изпълняват от компилатора.
В езика C++ се използват два вида коментари:
- Едноредови
```c++
// Това е коментар
```
- Многоредови
```c++
/* текст */
```

## 3. Основни управляващи конструкции условни оператори, оператори за цикъл
Чрез тях се постигат разклонения в логиката, повторения на действия и промяна на нормалния последователен поток на изпълнение.

### Условни конструкции
- if конструкция - Изпълнява блок от код само ако условието е вярно.
    ```c++
    if (условие) {
        // инструкции при вярно условие
    }

    // Пример
    if (x > 0) {
        std::cout << "Положително число";
    }
    ```
- if-else конструкция - Изпълнява един блок, ако условието е вярно, и друг – ако е невярно.
    ```c++
    if (условие) {
        // инструкции при вярно условие
    } else {
        // инструкции при невярно условие
    }

    // Пример
    if (x % 2 == 0) {
        std:: cout << "Четно число";
    } else {
        std::cout << "Нечетно число";
    }
    ```
- if - else if - else - Използва се, когато има повече от две възможности (разклонения).
    ```c++
    if (условие1) {
        // код при условие1
    } else if (условие2) {
        // код при условие2
    } else {
        // код, ако никое от горните не е вярно
    }

    // Пример
    if (x < 0) {
        std::cout << "Отрицателно";
    } else if (x == 0) {
        std::cout << "Нула";
    } else {
        std::cout << "Положително";
    }
    ```
- switch - Използва се, когато трябва да се избере между множество цели стойности на дадена променлива (обикновено цяло число или символ).
    ```c++
    switch (израз) {
    case стойност1:
        // инструкции
        break;
    case стойност2:
        // инструкции
        break;
    ...
    default:
        // инструкции при липса на съвпадение
    }

    // Пример
    int day = 3;
    switch (day) {
        case 1: cout << "Понеделник"; break;
        case 2: cout << "Вторник"; break;
        case 3: cout << "Сряда"; break;
        default: cout << "Невалиден ден";
    }
    ```
- Тернарен оператор (?:) - Тернарният оператор е съкратена форма на условна конструкция if-else, използвана за присвояване на стойност или избор между две алтернативи в едно изречение (израз). Нарича се тернарен, защото работи с три операнда.
    ```c++
    условие ? стойност_ако_е_вярно : стойност_ако_е_невярно;
    
    // Пример
    int a = 5, b = 10;
    int max = (a > b) ? a : b;
    ```

### Цикли
Циклите се използват за многократно изпълнение на един и същи блок от код, докато дадено условие е вярно.
- for цикъл - Подходящ, когато броят на итерациите е известен предварително.
    ```c++
    for (инициализация; условие; стъпка) {
        // инструкции
    }

    // Пример
    for (int i = 0; i < 5; i++) {
        std::cout << i << " ";
    }
    ```
- while цикъл - Използва се, когато броят на повторенията не е известен предварително, а зависи от дадено условие.
    ```c++
    while (условие) {
        // инструкции
    }

    // Пример
    int x = 1;
    while (x <= 5) {
        cout << x << " ";
        x++;
    }
    ```
- do...while цикъл - Изпълнява кода поне веднъж, дори ако условието не е вярно в началото. Проверява условието след първото изпълнение.
    ```c++
    do {
        // инструкции
    } while (условие);

    // Пример
    int x = 0;
    do {
        cout << x << " ";
        x++;
    } while (x < 3);
    ```

### Оператори за контрол на потока
Тези оператори управляват потока на изпълнение вътре в цикли или функции.
- break - Прекъсва изпълнението на най-близкия цикъл или switch конструкция.
    ```c++
    for (int i = 0; i < 10; i++) {
        if (i == 5) break;
        cout << i << " ";
    }
    ```
- continue - Прекъсва текущата итерация и преминава към следващата.
    ```c++
    for (int i = 0; i < 5; i++) {
        if (i == 2) continue;
        cout << i << " ";
    }
    ```
- return - Прекъсва изпълнението на функция и връща стойност (или просто излиза от void функция).
    ```c++
    int max(int a, int b) {
        if (a > b) return a;
        else return b;
    }
    ```

## 4. Променливи
Променливата е именувана област от паметта, предназначена за съхраняване на стойност от определен тип. Чрез името на променливата програмата може да достъпва, използва и променя съхранената стойност.

Всяка променлива се характеризира с:
 - име — идентификатор, чрез който променливата се използва в програмата;
 - тип — определя вида на стойностите, които могат да се съхраняват, размера на заетата памет и допустимите операции;
 - стойност — конкретните данни, съхранени в променливата;
 - обхват — частта от програмата, в която променливата е достъпна;
 - време на живот — периодът, през който променливата съществува в паметта.

Общият вид на дефиниция на променлива е:
```c++
<тип> <име>;
```
Например:
```c++
int age;
double price;
char symbol;
bool isValid;
```

Променлива може да бъде създадена и с начална стойност:

```c++
int age = 20;
double price = 15.50;
char symbol = 'A';
bool isValid = true;
```

Това се нарича инициализация.

### Локални променливи
Локалните променливи са променливи, декларирани вътре във функция или блок от инструкции. Те са достъпни само в рамките на блока, в който са дефинирани.

```c++
void example() {
    int x = 10;
    std::cout << x;
}
```

В този пример променливата x е локална за функцията example. Тя не може да бъде използвана извън тялото на функцията.

Локалните автоматични променливи се създават при влизане в блока и се унищожават при излизане от него. Ако такава променлива не бъде инициализирана, нейната стойност е неопределена.

```c++
void example() {
    int x; // неинициализирана локална променлива
}
```

### Глобални променливи
Глобалните променливи се дефинират извън всички функции. Те са достъпни от различни части на програмата след мястото на тяхната декларация.

```c++
int counter = 0;

void increment() {
    counter++;
}
```

В този пример counter е глобална променлива и може да бъде използвана във функцията increment.

Глобалните променливи съществуват през цялото време на изпълнение на програмата. Ако не бъдат изрично инициализирани, те получават начална нулева стойност.

Въпреки че глобалните променливи са удобни в някои случаи, прекомерната им употреба не се препоръчва. Те могат да затруднят проследяването на програмната логика, защото стойността им може да бъде променяна от различни функции.

### Инициализация на променливи
Инициализацията представлява задаване на начална стойност на променлива в момента на нейното създаване.

```c++
int x = 5;
double pi = 3.14;
char letter = 'A';
```
Инициализацията е важна, защото осигурява предвидима начална стойност. Това намалява риска от грешки, свързани с използването на неинициализирани променливи.

В C++ съществуват различни форми на инициализация, например:

```c++
int a = 5;    // копираща инициализация
int b(5);     // директна инициализация
int c{5};     // списъчна инициализация
```

### Оператор за присвояване

Операторът за присвояване служи за задаване на нова стойност на вече съществуваща променлива. В повечето езици за програмиране, включително C++, той се означава със знака =.

Общият вид е:

```c++
променлива = израз;
```

Първо се изчислява стойността на израза отдясно, след което тя се записва в променливата отляво.
```c++
int x;
x = 10;

int y = 5;
y = y + 3;
```

При инициализацията променливата получава стойност в момента на създаването си. При присвояването вече съществуваща променлива получава нова стойност.

Стойността, която се присвоява, трябва да бъде съвместима с типа на променливата. Ако типовете не съвпадат, може да се получи грешка при компилация или неявно преобразуване на типа.

```c++
int n;
n = 3.14; // стойността се преобразува до 3
```

Освен обикновения оператор =, често се използват и съкратени оператори за присвояване:
```c++
x += 5;  // x = x + 5
x -= 2;  // x = x - 2
x *= 3;  // x = x * 3
x /= 4;  // x = x / 4
```
Те съчетават аритметична операция и присвояване на резултата обратно в същата променлива.

## 5. Функции и процедури

Функциите и процедурите са основни средства за структуриране на програмата в процедурното програмиране. Чрез тях дадена задача се разделя на по-малки подзадачи, които могат да бъдат реализирани, тествани и използвани многократно.

Функцията е именувана подпрограма, която изпълнява определена задача и може да върне резултат. Процедурата е подпрограма, която извършва действие, но не връща стойност. В езика C++ няма отделна конструкция „процедура“ — тя се реализира чрез функция с тип на връщане void.

Общият вид на функция е:
```c++
<тип_на_връщане> <име_на_функция>(<списък_от_параметри>) {
    // тяло на функцията
}
```

Пример за функция, която връща стойност:
```c++
int sum(int a, int b) {
    return a + b;
}
```

Пример за процедура във вид на void функция:
```c++
void printMessage() {
    std::cout << "Hello";
}
```

Функцията се състои от няколко основни елемента:

- име — служи за идентифициране и извикване на функцията;
- тип на връщаната стойност — определя какъв резултат връща функцията;
- формални параметри — стойности, които функцията приема при извикване;
- тяло на функцията — блок от инструкции, които се изпълняват при извикване;
- оператор return — прекратява изпълнението на функцията и връща резултат, ако функцията не е от тип void.

Сигнатурата на функцията включва нейното име и списъка от параметри — техния брой, типове и ред. В C++ типът на връщаната стойност е част от декларацията, но сам по себе си не е достатъчен за различаване на претоварени функции.

### Параметри и видове параметри
Параметрите са средство за предаване на данни към функции и процедури. Те позволяват една и съща функция да работи с различни входни стойности.

Различават се:
 - формални параметри — параметрите, записани в дефиницията на функцията;
 - фактически параметри, наричани още аргументи — конкретните стойности или променливи, които се подават при извикване на функцията.

Пример:
```c++
int multiply(int x, int y) {
    return x * y;
}

int result = multiply(4, 5);
```
В този пример x и y са формални параметри, а 4 и 5 са фактически параметри.

Според предназначението си параметрите могат да бъдат:

 - входни параметри — използват се само за подаване на стойности към функцията;
 - изходни параметри — чрез тях функцията връща резултат чрез промяна на подадена променлива;
 - входно-изходни параметри — използват се както за подаване на начална стойност, така и за връщане на променена стойност.

### Предаване на параметри по име и по стойност.
 - При предаване по стойност на функцията се подава копие на фактическия параметър. Промените върху параметъра вътре във функцията не засягат оригиналната променлива.
    ```c++
    void change(int x) {
    x = 10;
    }   

    int a = 5;
    change(a);
    ```
    След извикването стойността на a остава 5, защото функцията е работила с копие.
    Предаването по стойност е подходящо, когато функцията трябва само да използва дадена стойност, без да я променя извън своето тяло.
 - При предаване по име на подпрограмата не се предава директно стойността на аргумента, а самият израз или име, чрез което този аргумент се изчислява. При всяко използване на параметъра в тялото на подпрограмата той се оценява в контекста на извикващата програма.

    Този механизъм е характерен за някои по-стари езици за програмиране, например Algol. В C++ предаването по име не е стандартен механизъм. В C++ обичайните начини за предаване са по стойност, по референция и чрез указател.

- По референция (by reference)
    Предава се псевдоним на оригиналния аргумент. Промените влияят директно на извънфункционалната стойност.
    ```c++
    void foo(int& x) { x = 10; } // x се променя извън функцията
    ```
- По указател (pointer)
    Предава се адрес на стойност. Изисква разименуване, но работи подобно на референция.
    ```c++
    void foo(int* x) { *x = 10; } // отново променя оригиналната стойност
    ```

Пояснение: Указателите са променливи, които съхраняват адреси на други променливи в паметта и позволяват индиректен достъп до тях. Референциите предоставят по-безопасен начин за достъп до променливи по адрес. Използват се при предаване на параметри и динамично заделяне на памет.

### Типове и проверка за съответствие на тип
В езиците със статична типизация, какъвто е C++, типът на всеки параметър се задава още при дефиниране или декларация на функцията. При извикване на функцията компилаторът проверява дали броят и типовете на подадените аргументи съответстват на формалните параметри.

Пример:

```c++
void printInt(int x) {
    std::cout << x;
}

printInt(5);
```

Тук извикването е коректно, защото функцията очаква параметър от тип int.

Ако се подаде стойност от друг тип, компилаторът може да извърши неявно преобразуване, ако такова е допустимо:
```c++
printInt(3.14);
```

В този случай стойността 3.14 може да бъде преобразувана до 3, което води до загуба на информация. Затова е важно типовете на аргументите да съответстват на типовете на параметрите.

При претоварване на функции проверката на типовете е особено важна, защото компилаторът избира коя функция да извика според броя, типовете и реда на подадените аргументи.

```c++
void print(int x);
void print(double x);
```

Ако се извика print(5), ще бъде избрана версията с параметър int, а при print(5.5) ще бъде избрана версията с параметър double.

## 6. Символни низове.
Символният низ е крайна последователност от символи, използвана за представяне и обработка на текстова информация. Низовете се използват при работа с имена, съобщения, потребителски вход, файлови пътища и други текстови данни.

В езика C++ символни низове могат да се представят по два основни начина:

 - чрез C-низове — масиви от символи, завършващи със специален символ '\0';
 - чрез std::string — клас от стандартната библиотека, който предоставя по-удобен и безопасен интерфейс за работа с текст.

### Представяне в паметта

В C++ типът char се използва за съхраняване на един символен код. Размерът на char е един байт. Всеки символ се представя чрез числова стойност според използваната кодова таблица или кодиране. При класическите символи често се използва ASCII, а при съвременни приложения — кодировки като UTF-8.

C-низът се представя като последователност от символи, разположени в последователни клетки на паметта. Краят на низа се означава със специалния нулев символ '\0'.

```c++
char word[] = {'h', 'e', 'l', 'l', 'o', '\0'};
```
Същият низ може да бъде записан и така:
```c++
char word[] = "hello";
```

В този случай в паметта се съхраняват символите 'h' 'e' 'l' 'l' 'o' '\0', следователно низът "hello" има дължина 5 символа, но заема 6 елемента в масива, защото последният елемент е крайният символ '\0'.

При std::string низът се представя чрез обект, който управлява последователност от символи и предоставя готови методи за работа с нея:
```c++
std::string text = "hello";
```

За разлика от C-низовете, при std::string не е необходимо програмистът ръчно да следи крайния символ и размера на заделената памет при обикновена употреба.

### Основни операции със символни низове
| Операция            | Функция                | Описание                                       |
| ------------------- | ---------------------- | ---------------------------------------------- |
| Дължина             | `strlen(str)`          | Връща броя символи без `'\0'`                  |
| Копиране            | `strcpy(dst, src)`     | Копира съдържанието на `src` в `dst`           |
| Конкатенация        | `strcat(dst, src)`     | Добавя `src` в края на `dst`                   |
| Сравнение           | `strcmp(s1, s2)`       | Сравнява лексикографски; връща 0, ако са равни |
| Търсене             | `strchr(str, ch)`      | Намира първото срещане на символ `ch` в `str`  |
| Извличане на подниз | `strncpy(dst, src, n)` | Копира до `n` символа                          |

Забележка: Масивът представлява последователност от елементи от един и същи тип, подредени в непрекъснат блок от памет. Всеки елемент има индекс, започващ от 0, което позволява достъп чрез квадратни скоби: array[index].
