# Tема 12: Процедурно програмиране - указатели, масиви и рекурсия

Анотация:
- Указатели и указателна аритметика.
- Едномерни и многомерни масиви - Основни операции с масиви - индексиране; Сортиране и търсене в едномерен масив - основни алгоритми.
- Символни низове. Представяне в паметта. Основни операции със символни.
- Рекурсия - пряка и косвена рекурсия, линейна и разклонена рекурсия.

## Въведение

Процедурното програмиране организира програмата като последователност от инструкции, обединени във функции и процедури, които обработват данни чрез променливи и управляващи конструкции.

Всяка променлива заема определено място в паметта и има адрес. Указателите позволяват работа с такива адреси и осигуряват възможност за индиректен достъп до данни. Масивите представляват последователности от еднотипни елементи, разположени в паметта, а символният низ може да се разглежда като специален случай на масив от символи. Рекурсията, от своя страна, е метод за решаване на задачи чрез свеждането им до по-малки еднотипни подзадачи.

Разглеждането на тези понятия е важно, защото те са основа за реализиране на по-сложни алгоритми, структури от данни и ефективна обработка на информация в процедурното програмиране.

## 1. Указатели и видове указатели
Паметта в един компютър се състои от поредица байтове, които са индексирани чрез адреси. Указателят е променлива, която съхранява адреса в паметта на друга променлива или обект. Чрез указатели се осъществява индиректен достъп до данни, т.е. достъп не чрез самото име на променливата, а чрез адреса, на който тя се намира. 

Указателите са особено важни при работа с масиви, динамична памет, функции и структури от данни.

Общият синтаксис за деклариране на указател е:

```c++
<тип>* <име_на_указател>;
```

Например:

```cpp
int* p;
double* q;
char* c;
```

Тук `p` е указател към стойност от тип `int`, `q` е указател към стойност от тип `double`, а `c` е указател към стойност от тип `char`.

Размерът на указателя зависи от архитектурата и модела на паметта. Обикновено при 32-битови системи указателят заема 4 байта, а при 64-битови системи — 8 байта. Този размер не зависи от типа на данната, към която указателят сочи, а от начина, по който системата адресира паметта.

---

### Оператор за вземане на адрес

Адресът на дадена променлива се получава чрез унарния оператор `&`. `&` може да се прилага само върху lvalue данни — данни които имат адрес в паметта. Не може да се прилага върху rvalue данни — стойности, които нямат адрес (напр. временни изрази).

```cpp
int num = 42;
int* ptr = &num;
```

В този пример `ptr` съхранява адреса на променливата `num`. Типът на указателя трябва да съответства на типа на променливата, към която сочи.

```cpp
float pi = 3.14f;
float* piPtr = &pi;     // правилно
int* wrongPtr = &pi;    // грешка: несъвместими типове
```

---

### Дереференция на указател

Операторът `*` се използва за достъп до стойността, която се намира на адреса, съхранен в указателя. Това действие се нарича дереференция.

```cpp
int num = 10;
int* ptr = &num;

std::cout << ptr;   // отпечатва адреса на num
std::cout << *ptr;  // отпечатва стойността на num, т.е. 10
```

Чрез дереференция може не само да се чете, но и да се променя стойността на променливата, към която указателят сочи.

```cpp
int num = 10;
int* ptr = &num;

*ptr = 20; // променя стойността на num
```

След изпълнението на този код стойността на `num` е `20`.

---

### Видове указатели

- Обикновен указател

Обикновеният указател съхранява адреса на променлива от конкретен тип.

```cpp
int x = 5;
int* p = &x;
```

Тук `p` сочи към променливата `x`.

---

- Нулев указател

Нулевият указател е указател, който не сочи към валиден обект. В C++ за тази цел се използва pointer literal със специална стойност за нулев адрес `nullptr`.

```cpp
int* p = nullptr;
```

Използването на `nullptr` е добра практика, когато указателят все още не сочи към конкретна променлива или когато искаме ясно да покажем, че той не съдържа валиден адрес.

Преди дереференция е добре да се провери дали указателят не е нулев:

```cpp
if (p != nullptr) {
    std::cout << *p;
}
```

Дереференцирането на `nullptr` води до неопределено поведение и може да причини срив на програмата.

---

- Указател към указател

Указател към указател е променлива, която съхранява адреса на друг указател.

```cpp
int x = 5;
int* p = &x;
int** pp = &p;

std::cout << **pp; // отпечатва 5
```

В този пример `p` сочи към `x`, а `pp` сочи към `p`. Чрез `**pp` се достига до стойността на `x`.

---

- Указател към константа

Указател към константа е указател, чрез който не може да се променя стойността на обекта, към който сочи.

```cpp
int x = 5;
const int* p = &x;

// *p = 10; // грешка
p = nullptr; // позволено е указателят да сочи другаде
```

Тук не може да се променя стойността чрез `p`, но самият указател може да бъде пренасочен.

---

- Константен указател

Константният указател е указател, който след инициализация не може да бъде пренасочен към друг адрес.

```cpp
int x = 5;
int y = 10;

int* const p = &x;

*p = 20; // позволено е да се промени стойността на x
// p = &y; // грешка: p не може да бъде пренасочен
```

Тук самият указател е константен, но стойността, към която сочи, може да бъде променяна.

---

- Константен указател към константа

При този вид указател не може да се променя нито самият указател, нито стойността, към която той сочи.

```cpp
int x = 5;
const int* const p = &x;

// *p = 10;      // грешка
// p = nullptr;  // грешка
```

---

- Указател от тип `void*`

Указателят от тип `void*` е общ указател, който може да съхранява адрес на обект от различен тип.

```cpp
int x = 5;
void* p = &x;
```

Тъй като `void*` не съдържа информация за конкретния тип на обекта, той не може да бъде директно дереференциран. Преди достъп до стойността трябва да бъде преобразуван към подходящ тип указател.

```cpp
int x = 5;
void* p = &x;

int* intPtr = static_cast<int*>(p);
std::cout << *intPtr;
```

---

### Основни операции при указателна аритметика

При аритметика, стойността на указателя се увеличава/намалява със стойността на sizeof(тип). Формално, ако `p` e указател от тип `T`, то инкрементирането на `T*` с `i` e еквивалентно на `p + i * sizeof T`, където `sizeof Т` връща броя байтове, необходими за съхраняване на данна от тип `Т`

- Събиране на указател с цяло число

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p = arr;

std::cout << *(p + 3); // 4
```

Тук `p + 3` сочи към елемента с индекс `3`.

---

- Изваждане на цяло число от указател

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p = &arr[4];

std::cout << *(p - 2); // 3
```

Тук `p - 2` сочи към елемента с индекс `2`.

---

- Инкрементиране и декрементиране на указател

```cpp
int arr[3] = {10, 20, 30};
int* p = arr;

std::cout << *p; // 10

p++;
std::cout << *p; // 20

p--;
std::cout << *p; // 10
```

Операцията `p++` премества указателя към следващия елемент, а `p--` го връща към предишния.

---

- Разлика между два указателя

Когато два указателя сочат към елементи от един и същ масив, може да се изчисли разликата между тях. Резултатът показва колко елемента има между двата адреса, а не колко байта.

```cpp
int arr[5] = {1, 2, 3, 4, 5};

int* p1 = &arr[1];
int* p2 = &arr[4];

std::cout << p2 - p1; // 3
```

Тук резултатът е `3`, защото между елементите с индекси `1` и `4` има разлика от три позиции.

---

- Логически сравнения
При прилагането на оператори за сравнение към указатели се сравняват стойностите на адресите, които те съдържат. Два указателя са равни, ако сочат към един и същ адрес (т.е. към една и съща променлива).

```c++
int x = 5, y = 10;
int* px = &x;
int* py = &y;

if (px != py)
    std::cout << "Указателите сочат към различни променливи";
```
---

- Ограничения на указателната аритметика

Указателната аритметика трябва да се използва внимателно. Тя е коректна основно когато указателите сочат към елементи от един и същ масив или към позицията непосредствено след последния елемент на масива.

Извъндиапазонен достъп, например дереференция на указател извън границите на масива, води до неопределено поведение.

```cpp
int arr[3] = {1, 2, 3};

int* p = arr + 3; // позволено е да сочи след последния елемент
// std::cout << *p; // грешка: не трябва да се дереференцира
```

Събирането на два указателя няма смисъл и не е позволено. Разрешени са операции като добавяне или изваждане на цяло число от указател и изваждане на два указателя, когато са свързани с един и същ масив.

---

- Потенциални грешки при работа с указатели

- **неинициализиран указател** — указател, който съдържа неопределена стойност;
- **нулев указател** — указател със стойност `nullptr`, който не трябва да се дереференцира;
- **висящ указател** — указател, който сочи към памет, която вече е освободена;
- **изтичане на памет** — динамично заделена памет, която не е освободена;
- **дереференция на невалиден указател** — достъп до памет, която не принадлежи на програмата.

Добра практика е указателите да се инициализират при създаване, а след освобождаване на динамична памет да се задават на `nullptr`.

```cpp
int* p = nullptr;
```

## 2. Едномерни и многомерни масиви

Масивът е съставна структура от данни, която съдържа множество елементи от един и същи тип, разположени последователно в паметта. Всеки елемент на масива се достъпва чрез индекс, като индексите започват от 0.

```cpp
<тип> <име>[брой_елементи];
```

Например:

```cpp
int arr[5];
```

Тук `arr` е масив от 5 елемента от тип `int`.

---

### Едномерни масиви
- Деклариране:
    ```c++
    int arr[5] = {1, 2, 3, 4, 5};
    ```
- Достъп:
    ```c++
    int x = arr[2];  // x = 3
    ```
---

### Индексиране на масиви

Индексирането е основна операция при работа с масиви. Чрез него се достъпва конкретен елемент по неговата позиция.
Ако масивът има `n` елемента, валидните индекси са от `0` до `n - 1`. Тъй като индексите започват от 0, елементът `arr[3]` е четвъртия елемент на масива.

```cpp
int arr[4] = {10, 20, 30, 40};

std::cout << arr[0]; // 10
std::cout << arr[3]; // 40
```

Достъпът извън границите на масива води до неопределено поведение.

```cpp
int arr[4] = {10, 20, 30, 40};

// arr[4] е невалиден достъп, защото последният валиден индекс е 3
```

Затова при обхождане на масив е важно индексът да остава в допустимите граници.

```cpp
for (int i = 0; i < 4; i++) {
    std::cout << arr[i] << " ";
}
```
---

### Връзка между масиви и указатели

В C++ масивите са тясно свързани с указателите. В много изрази името на масива се преобразува до указател към първия му елемент.

```cpp
int arr[3] = {10, 20, 30};

std::cout << arr[0];      // 10
std::cout << *(arr + 0);  // 10

std::cout << arr[1];      // 20
std::cout << *(arr + 1);  // 20
```

Изразът:

```cpp
arr[i]
```

е еквивалентен на:

```cpp
*(arr + i)
```

Това означава, че индексирането на масива може да се разглежда като форма на указателна аритметика.

Важно е обаче да се отбележи, че самият масив не е указател. Масивът представлява последователен блок от памет, а в определени изрази името му се преобразува до адреса на първия елемент.

---

### Многомерни масиви

Многомерният масив е масив, чиито елементи също могат да се разглеждат като масиви. Най-често използван е двумерният масив, който логически се представя като таблица от редове и колони.

Общ вид на двумерен масив:

```cpp
<тип> <име>[брой_редове][брой_колони];
```

Пример:

```cpp
int matrix[3][4];
```

Тук `matrix` е двумерен масив с 3 реда и 4 колони.

Достъпът до елемент се извършва чрез два индекса:

```cpp
matrix[1][2] = 7;
```

Това означава: елементът на ред с индекс `1` и колона с индекс `2` получава стойност `7`.

При двумерен масив с `rows` реда и `cols` колони валидните индекси са:

```cpp
0 <= i < rows
0 <= j < cols
```

където `i` е индексът на реда, а `j` е индексът на колоната.

---

### Представяне на двумерен масив в паметта

В C++ двумерните масиви се съхраняват последователно в паметта по редове. Това означава, че първо се разполагат всички елементи от първия ред, след това всички елементи от втория ред и т.н.

Например масивът:

```cpp
int matrix[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};
```

логически изглежда така:

```text
1 2 3
4 5 6
```

а в паметта елементите са подредени така:

```text
matrix[0][0], matrix[0][1], matrix[0][2], matrix[1][0], matrix[1][1], matrix[1][2]
```

Тоест последователността в паметта е:

```text
1, 2, 3, 4, 5, 6
```
---

### Обхождане на двумерен масив

Двумерните масиви обикновено се обхождат чрез вложени цикли. Външният цикъл преминава през редовете, а вътрешният — през колоните.

```cpp
int matrix[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};

for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 3; j++) {
        std::cout << matrix[i][j] << " ";
    }
    std::cout << std::endl;
}
```

---

### Масиви като параметри на функции

Когато едномерен масив се подава като параметър на функция, той не се копира. Във функцията се предава адресът на първия елемент на масива. Това означава, че функцията работи директно с оригиналния масив и всички промени в него ще се отразят и извън функцията.

```cpp
void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        std::cout << arr[i] << " ";
    }
}
```

Следните записи са еквивалентни като параметър на функция:

```cpp
void func(int arr[]);
void func(int arr[10]);
void func(int* arr);
```

И в трите случая параметърът се разглежда като указател към `int`.

При многомерни масиви също се предава указател, но трябва да се уточнят размерите на всички измерения, с изключение на първото:

```cpp
void printMatrix(int matrix[][3], int rows) {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < 3; j++) {
            std::cout << matrix[i][j] << " ";
        }
        std::cout << std::endl;
    }
}
```

Броят на колоните трябва да бъде известен, за да може компилаторът правилно да изчислява адреса на всеки елемент.

### Сортиране в едномерен масив

Сортирането представлява подреждане на елементите на масив по определен критерий — например във възходящ или низходящ ред. При процедурното програмиране често се разглеждат основни алгоритми за сортиране, които са лесни за разбиране и реализация.

- Bubble Sort

Bubble Sort сравнява съседни елементи и ги разменя, ако са в грешен ред. След всяко преминаване най-големият елемент постепенно се премества към края на масива.

```cpp
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - i - 1; j++) {
        if (arr[j] > arr[j + 1]) {
            std::swap(arr[j], arr[j + 1]);
        }
    }
}
```

Характеристики:
- лесен за разбиране, подходящ за малки масиви, неефективен при голям брой елементи, времева сложност: `O(n²)`.

---

- Selection Sort

Selection Sort намира най-малкия елемент в несортираната част на масива и го поставя на правилната позиция.

```cpp
for (int i = 0; i < n - 1; i++) {
    int minIndex = i;

    for (int j = i + 1; j < n; j++) {
        if (arr[j] < arr[minIndex]) {
            minIndex = j;
        }
    }

    std::swap(arr[i], arr[minIndex]);
}
```

Характеристики:

- прост алгоритъм, прави сравнително малко размени, подходящ за малки масиви, времева сложност: `O(n²)`.

---

- Insertion Sort

Insertion Sort изгражда сортирана част на масива, като всеки следващ елемент се вмъква на правилното място в нея.

```cpp
for (int i = 1; i < n; i++) {
    int key = arr[i];
    int j = i - 1;

    while (j >= 0 && arr[j] > key) {
        arr[j + 1] = arr[j];
        j--;
    }

    arr[j + 1] = key;
}
```

Характеристики:

- ефективен при малки или почти сортирани масиви, лесен за реализация, времева сложност в най-лошия случай: `O(n²)`, при почти сортиран масив може да работи близо до `O(n)`.

---

### Търсене в едномерен масив

Търсенето е операция, при която се проверява дали дадена стойност се съдържа в масива и, ако се съдържа, се намира нейният индекс.

- Линейно търсене

Линейното търсене последователно проверява всеки елемент на масива, докато намери търсената стойност или докато достигне края на масива.

```cpp
int linearSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) {
            return i;
        }
    }

    return -1;
}
```

Ако елементът бъде намерен, функцията връща неговия индекс. Ако не бъде намерен, връща `-1`.

Характеристики:
- работи както при сортирани, така и при несортирани масиви, подходящ за малки масиви, времева сложност: `O(n)`.

---

- Двоично търсене

Двоичното търсене е по-ефективен алгоритъм, но изисква масивът предварително да бъде сортиран. То сравнява търсената стойност със средния елемент на масива и според резултата продължава търсенето само в лявата или дясната половина.

```cpp
int binarySearch(int arr[], int n, int target) {
    int left = 0;
    int right = n - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;
}
```

Характеристики:
- изисква сортиран масив, много по-ефективен от линейното търсене при големи масиви, времева сложност: `O(log n)`.

## 3. Символни низове.
Символният низ е крайна последователност от символи, използвана за представяне и обработка на текстова информация. Низовете се използват при работа с имена, съобщения, потребителски вход, файлови пътища и други текстови данни.

В езика C++ символни низове могат да се представят по два основни начина:

 - чрез C-низове — масиви от символи, завършващи със специален символ '\0';
 - чрез std::string — клас от стандартната библиотека, който предоставя по-удобен и безопасен интерфейс за работа с текст.

### Представяне в паметта

В C++ типът char се използва за съхраняване на един символен код. Размерът на char е един байт. Всеки символ се представя чрез числова стойност според използваната кодова таблица или кодиране. При класическите символи често се използва ASCII, а при съвременни приложения — кодировки като UTF-8.

C-низът се представя като последователност от символи, разположени в последователни клетки на паметта. Краят на низа се означава със специалния нулев символ '\0'.

```c++
char word[] = {'h', 'e', 'l', 'l', 'o', '\0'};
```
Същият низ може да бъде записан и така:
```c++
char word[] = "hello";
```

В този случай в паметта се съхраняват символите 'h' 'e' 'l' 'l' 'o' '\0', следователно низът "hello" има дължина 5 символа, но заема 6 елемента в масива, защото последният елемент е крайният символ '\0'.

При std::string низът се представя чрез обект, който управлява последователност от символи и предоставя готови методи за работа с нея:
```c++
std::string text = "hello";
```

За разлика от C-низовете, при std::string не е необходимо програмистът ръчно да следи крайния символ и размера на заделената памет при обикновена употреба.

### Основни операции със символни низове
| Операция            | Функция                | Описание                                       |
| ------------------- | ---------------------- | ---------------------------------------------- |
| Дължина             | `strlen(str)`          | Връща броя символи без `'\0'`                  |
| Копиране            | `strcpy(dst, src)`     | Копира съдържанието на `src` в `dst`           |
| Конкатенация        | `strcat(dst, src)`     | Добавя `src` в края на `dst`                   |
| Сравнение           | `strcmp(s1, s2)`       | Сравнява лексикографски; връща 0, ако са равни |
| Търсене             | `strchr(str, ch)`      | Намира първото срещане на символ `ch` в `str`  |
| Извличане на подниз | `strncpy(dst, src, n)` | Копира до `n` символа                          |

## 4. Рекурсия
Рекурсията е техника в програмирането, при която функция извиква сама себе си пряко или косвено с цел решаване на задача чрез свеждането ѝ до по-малки задачи от същия вид.

В теорията на изчислимостта е доказано, че всяка рекурсивна функция може да бъде преобразувана в итеративна, но рекурсията често е по-интуитивен и удобен начин при задачи, които имат естествено рекурсивна структура — например изчисляване на факториел, ред на Фибоначи, обхождане на дървета, обработка на вложени структури и други.

Всяка правилно построена рекурсивна функция трябва да има два основни елемента:

- **базов случай (дъно)** — условие, при което рекурсията спира;
- **рекурсивен случай** — частта, в която функцията извиква себе си с по-прост или по-малък аргумент.

Ако липсва базов случай или ако рекурсивното извикване не се доближава до него, може да се получи безкрайна рекурсия и препълване на стека.

---

### Изпълнение на рекурсивна функция

При всяко извикване на функцията в паметта се създава нова стекова рамка. Тя съдържа параметрите, локалните променливи и информацията, необходима за връщане към предишното извикване.

Изпълнението на рекурсията може да се разглежда в два етапа:

1. **Разгъване на рекурсията** — функцията се извиква многократно, докато се достигне базовият случай.
2. **Свиване на рекурсията** — резултатите започват да се връщат обратно към предишните извиквания.

Дълбочината на рекурсията е броят на вложените рекурсивни извиквания до достигане на базовия случай. Ако дълбочината стане прекалено голяма, може да се получи препълване на стека.

---

### Пример за рекурсия — факториел

Факториелът на числото `n` се дефинира така:

```text
n! = n * (n - 1)!
0! = 1
```

Тази дефиниция е рекурсивна, защото факториелът на `n` се изразява чрез факториела на по-малко число.

```cpp
int factorial(int n) {
    if (n == 0) {
        return 1; // базов случай
    }

    return n * factorial(n - 1); // рекурсивен случай
}
```

При извикване на `factorial(4)` се получава:

```text
factorial(4) = 4 * factorial(3)
factorial(3) = 3 * factorial(2)
factorial(2) = 2 * factorial(1)
factorial(1) = 1 * factorial(0)
factorial(0) = 1
```

След достигане на базовия случай започва връщането на резултатите назад.

---

### Видове рекурсия
- Пряка рекурсия

Пряка рекурсия е налице, когато функцията извиква директно сама себе си.

```cpp
void countdown(int n) {
    if (n == 0) {
        return; // базов случай
    }

    std::cout << n << " ";
    countdown(n - 1); // пряко рекурсивно извикване
}
```

Тук функцията `countdown` директно извиква себе си с по-малка стойност на параметъра `n`.

---

- Косвена рекурсия

Косвена, или непряка, рекурсия има тогава, когато една функция извиква друга функция, която от своя страна отново извиква първата. Така се образува цикъл от взаимни извиквания.

```cpp
void functionB(int n);

void functionA(int n) {
    if (n <= 0) {
        return;
    }

    std::cout << "A " << n << std::endl;
    functionB(n - 1);
}

void functionB(int n) {
    if (n <= 0) {
        return;
    }

    std::cout << "B " << n << std::endl;
    functionA(n - 1);
}
```

При косвената рекурсия рекурсивният процес не се осъществява чрез директно извикване на същата функция, а чрез верига от извиквания между две или повече функции.

---

- Линейна рекурсия

Линейна рекурсия е рекурсия, при която във всяко изпълнение на функцията има най-много едно рекурсивно извикване.
Пример за линейна рекурсия е изчисляването на факториел:

```cpp
int factorial(int n) {
    if (n == 0) {
        return 1;
    }

    return n * factorial(n - 1);
}
```

---

- Разклонена рекурсия

Разклонена рекурсия е рекурсия, при която във всяко изпълнение на функцията може да има повече от едно рекурсивно извикване.
Класически пример е изчисляването на числа от реда на Фибоначи:

```cpp
int fibonacci(int n) {
    if (n <= 1) {
        return n; // базов случай
    }

    return fibonacci(n - 1) + fibonacci(n - 2);
}
```
