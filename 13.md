## Концептуални основи на обектно-ориентираното програмиране

Обектно-ориентираното програмиране (ООП) е програмна парадигма, при която програмната система се моделира чрез обекти. Обектите обединяват данни и операции върху тези данни и взаимодействат помежду си чрез извикване на методи.

За разлика от процедурното програмиране, при което основният акцент е върху последователността от инструкции и функции, в обектно-ориентираното програмиране основни единици са класовете и обектите. Класът описва общата структура и поведение на даден тип обекти, а обектът е конкретен екземпляр на този клас.

Обектно-ориентираният подход улеснява моделирането на сложни системи, тъй като позволява програмата да бъде разделена на логически самостоятелни компоненти. Това води до по-добра организация на кода, повторна употреба, по-лесна поддръжка и разширяемост.

Основните принципи на обектно-ориентираното програмиране са:

- **абстракция**;
- **капсулация**;
- **наследяване**;
- **полиморфизъм**.

### Абстракция
Абстракцията е принцип, при който се отделят съществените характеристики на даден обект или система, а несъществените детайли се скриват. Чрез абстракцията се описва какво прави даден обект, без задължително да се разкрива как точно е реализирано това поведение.

В C++ абстракцията се реализира основно чрез класове, които предоставят публичен интерфейс за работа с обектите и скриват вътрешната реализация.

### Капсулация
Капсулацията представлява обединяване на данни и методи, които работят с тях, в една обща структура — клас. Тя позволява вътрешното състояние на обекта да бъде защитено от неконтролиран външен достъп.

В C++ капсулацията се реализира чрез модификаторите за достъп `private`, `protected` и `public`. Обикновено данните на класа се правят `private`, а достъпът до тях се осъществява чрез публични методи.

### Наследяване

Наследяването е механизъм, чрез който един клас може да бъде създаден на основата на друг вече съществуващ клас. Съществуващият клас се нарича базов клас, а новият клас — производен клас.

Чрез наследяване производният клас получава характеристики и поведение от базовия клас, като може да добавя нови членове или да променя част от поведението. Така се постига повторна употреба на код и се изграждат йерархии от класове.

### Полиморфизъм

Полиморфизмът позволява един и същ интерфейс да се използва за обекти от различни класове, като конкретното поведение зависи от реалния тип на обекта.

В C++ полиморфизмът може да се проявява чрез претоварване на функции и оператори, както и чрез виртуални функции при наследяване. В тази тема полиморфизмът се споменава като основен принцип на ООП, но не се разглежда подробно.

## Абстракция със структури от данни

Както споменахме по-горе, при абстракцията потребителят работи с дадена структура чрез определен набор от операции, без да е необходимо да знае как точно са реализирани те. Например банкова сметка може да бъде разглеждана като абстракция. За потребителя на такава структура са важни основните операции, които могат да се извършват с нея:

- създаване на сметка с начално състояние;
- внасяне на сума;
- теглене на сума;
- проверка на текущия баланс;
- проверка дали дадена операция е допустима.

На това ниво не е необходимо да се знае как точно се съхранява балансът — дали като реално число, като цяло число в стотинки, дали се пази история на транзакциите. Тези подробности са част от вътрешната реализация. Важно е външният потребител да разполага с ясен интерфейс за работа със сметката. Например той трябва да може да внесе пари, да изтегли пари и да провери баланса, без да има директен достъп до вътрешните данни. Така банковата сметка може да се разглежда като структура от данни, която има състояние и набор от операции. Състоянието може да включва текущ баланс, номер на сметка или други данни, а операциите определят допустимите действия върху това състояние. В обектно-ориентираното програмиране тази идея се реализира чрез класове.

## Класове и обекти. Декларация на клас и декларация на обект

### Клас

Класът е основна конструкция в обектно-ориентираното програмиране. Той представлява описание, модел или шаблон, по който се създават обекти от даден тип. Класът обединява данни и операции върху тях в една обща програмна единица. Данните описват състоянието на обекта, а операциите описват неговото поведение. В C++ данните в класа се наричат **член-данни**, а функциите, дефинирани в класа, се наричат **методи** или **член-функции**.

Класът определя:

- какви данни ще съдържат обектите;
- какви операции могат да се извършват с тези данни;
- кои членове са достъпни отвън;
- кои членове са скрити като вътрешна реализация.

Общият вид на дефиниция на клас е:

```cpp
class ИмеНаКлас {
private:
    // скрити член-данни и методи

public:
    // публичен интерфейс
};
```

Например:

```cpp
class Person {
private:
    std::string name;
    int age;

public:
    void setName(const std::string& newName) {
        name = newName;
    }

    std::string getName() const {
        return name;
    }

    void setAge(int newAge) {
        if (newAge >= 0) {
            age = newAge;
        }
    }

    int getAge() const {
        return age;
    }
};
```

В този пример класът `Person` описва обекти, които имат име и възраст. Член-данните `name` и `age` описват състоянието на обекта, а методите `setName`, `getName`, `setAge` и `getAge` определят неговото поведение и начина за достъп до това състояние.

---

### Декларация и дефиниция на клас

В C++ може да се направи разлика между предварителна декларация на клас и дефиниция на клас. Предварителната декларация само съобщава на компилатора, че съществува клас с дадено име:

```cpp
class Person;
```

Тази декларация не описва член-данните и методите на класа. Тя е полезна, когато трябва да се използват указатели или референции към клас, който ще бъде дефиниран по-късно. Дефиницията на клас съдържа пълното описание на неговите членове:

```cpp
class Person {
private:
    std::string name;
    int age;

public:
    void setAge(int newAge);
    int getAge() const;
};
```
В C++ както `class`, така и `struct` могат да съдържат член-данни и методи. Основната разлика между тях е достъпът по подразбиране. При `class` членовете по подразбиране са `private`, а при `struct` по подразбиране са `public`.

Самото дефиниране на клас не създава конкретен обект. То само описва структурата и поведението на бъдещите обекти. Памет за нестатичните член-данни се заделя едва когато бъде създаден обект от този клас.

---

### Обект

Обектът е конкретен екземпляр, създаден по описанието на даден клас. Ако класът е модел, то обектът е реална променлива от този класов тип.

```cpp
Person p1;
Person p2;
```

Тук `p1` и `p2` са два различни обекта от класа `Person`. Всеки обект има собствено състояние, тоест собствено копие на нестатичните член-данни.

```cpp
p1.setName("Ivan");
p1.setAge(20);

p2.setName("Maria");
p2.setAge(25);
```

В този пример `p1` и `p2` са обекти от един и същ клас, но имат различни стойности за член-данните `name` и `age`. Методите са общо дефинирани в класа, но при извикване работят върху конкретния обект.

```cpp
std::cout << p1.getName(); // Ivan
std::cout << p2.getName(); // Maria
```

---

### Декларация на обект

Декларацията на обект създава променлива от класов тип.

```cpp
Person person;
```

Тук `person` е обект от класа `Person`.

При създаването на обект се заделя памет за неговите нестатични член-данни. Ако класът има конструктор, той се извиква автоматично при създаването на обекта. Обект може да бъде създаден и чрез конструктор с параметри, което ще разгледаме напред в темата.

---

### Достъп до членове на обект

Достъпът до публичните членове на обект се извършва чрез оператора точка `.`.

```cpp
Person p;

p.setName("Ivan");
p.setAge(20);

std::cout << p.getName();
```

Ако член-данните са обявени като `private`, те не могат да бъдат достъпвани директно извън класа.

```cpp
// p.age = 20; // грешка, ако age е private
```

Вместо това се използват публични методи, чрез които достъпът се контролира.

---
### Обхват на видимост

Класовете най-често се дефинират в глобален или namespace обхват, така че да могат да бъдат използвани от различни части на програмата. Възможно е клас да бъде дефиниран и локално — вътре във функция. В такъв случай той е достъпен само в рамките на тази функция.

## Основни видове конструктори. Управление на динамичната памет и ресурсите (RAII)

Конструкторът е специална член-функция на класа, която се извиква автоматично при създаване на обект. Основната му задача е да инициализира член-данните и да постави обекта във валидно начално състояние. Конструкторът има същото име като класа и няма тип на връщана стойност.

### Конструктор по подразбиране

Конструктор по подразбиране е конструктор, който може да бъде извикан без аргументи.

```cpp
class Point {
private:
    int x;
    int y;

public:
    Point() {
        x = 0;
        y = 0;
    }
};
```

Той се използва, когато обектът се създава без подаване на начални стойности:

```cpp
Point p;
```

Ако в класа не е дефиниран потребителски конструктор, компилаторът може автоматично да генерира конструктор по подразбиране. Ако обаче е дефиниран друг конструктор, например параметризиран, автоматичното генериране на конструктор по подразбиране може да отпадне.

Конструкторът по подразбиране е важен при създаване на масиви от обекти, защото при създаването на такъв масив всеки елемент трябва да бъде конструиран автоматично, без да му се подават индивидуални аргументи, което се случва чрез извикване на конструктора по подразбиране.

---

### Параметризиран конструктор

Параметризираният конструктор приема аргументи и позволява обектът да бъде създаден с конкретни начални стойности.

```cpp
class Point {
private:
    int x;
    int y;

public:
    Point(int xValue, int yValue) {
        x = xValue;
        y = yValue;
    }
};
```

Създаване на обект:

```cpp
Point p(2, 3);
```

Тук обектът `p` се създава със стойности `x = 2` и `y = 3`.

---

### Инициализиращ списък

В C++ член-данните могат да се инициализират чрез инициализиращ списък. Той се записва след параметрите на конструктора и преди тялото му.

```cpp
class Point {
private:
    int x;
    int y;

public:
    Point(int xValue, int yValue)
        : x(xValue), y(yValue) {
    }
};
```

Инициализиращият списък е предпочитан, защото извършва директна инициализация на член-данните, вместо първо да ги създава и след това да им присвоява стойности. Той е полезен при клас с `const` член-данна, защото трябва да бъде инициализирана чрез инициализиращ списък, защото е константа.

```cpp
class Student {
private:
    const int facultyNumber;

public:
    Student(int number)
        : facultyNumber(number) {
    }
};
```

---

### Копиращ конструктор

Копиращият конструктор създава нов обект като копие на вече съществуващ обект от същия клас.

Общ вид:

```cpp
ClassName(const ClassName& other);
```

Пример:

```cpp
class Point {
private:
    int x;
    int y;

public:
    Point(int xValue, int yValue)
        : x(xValue), y(yValue) {
    }

    Point(const Point& other)
        : x(other.x), y(other.y) {
    }
};
```

Копиращият конструктор се извиква, когато се създава нов обект от съществуващ обект:

```cpp
Point p1(2, 3);
Point p2 = p1;
```

В този пример `p2` се създава като копие на `p1`.

Копиращият конструктор е особено важен при класове, които управляват динамична памет или други ресурси. В такива случаи трябва да се осигури дълбоко копиране, за да не сочат два обекта към един и същ ресурс.

---

### Преместващ конструктор

Преместващият конструктор създава нов обект, като прехвърля ресурсите от временен или изтичащ обект. 

Общ вид:

```cpp
ClassName(ClassName&& other) noexcept;
```

Идеята е вместо ресурсът да се копира, собствеността върху него да се премести от един обект към друг. Преместващият конструктор е полезен при класове, които управляват динамична памет, файлове, мрежови връзки или други ресурси, чието копиране е скъпо.

---

### Деструктор

Деструкторът не е конструктор, но е специална член-функция, свързана с жизнения цикъл на обекта. Той се извиква автоматично при унищожаване на обекта.

Деструкторът има същото име като класа, но пред него се поставя символът `~`. Той няма тип на връщана стойност и не приема параметри.

```cpp
class MyClass {
public:
    ~MyClass() {
        // освобождаване на ресурси
    }
};
```

Основната задача на деструктора е да освободи ресурси, които обектът е придобил по време на своето съществуване — например динамична памет, файлове или други външни ресурси.

---

## Управление на динамичната памет и ресурсите

В C++ динамична памет може да бъде заделяна чрез `new` и освобождавана чрез `delete`.

```cpp
int* p = new int(5);

delete p;
p = nullptr;
```

При масиви се използват `new[]` и `delete[]`.

```cpp
int* arr = new int[10];

delete[] arr;
arr = nullptr;
```

Ако динамично заделената памет не бъде освободена, се получава изтичане на памет. Ако една и съща памет бъде освободена повече от веднъж, се получава грешка от тип double free. Ако указател сочи към вече освободена памет, той се нарича висящ указател. Поради тези рискове в C++ е важно ресурсите да се управляват чрез обекти, а не чрез ръчно извикване на `new` и `delete` на много места в програмата.

---

### RAII

RAII означава Resource Acquisition Is Initialization. Това е основен идиом в C++, при който придобиването на ресурс се свързва със създаването на обект, а освобождаването на ресурса — с унищожаването на обекта.

Идеята е:

- ресурсът се придобива в конструктора;
- ресурсът се освобождава в деструктора;
- когато обектът излезе от обхват, деструкторът се извиква автоматично;
- така се намалява рискът от изтичане на ресурси.

Пример:

```cpp
class IntArray {
private:
    int* data;
    int size;

public:
    IntArray(int s)
        : size(s) {
        data = new int[size];
    }

    ~IntArray() {
        delete[] data;
    }
};
```

При създаване на обект от класа `IntArray` в конструктора се заделя динамична памет. Когато обектът бъде унищожен, деструкторът автоматично освобождава тази памет.

```cpp
void example() {
    IntArray arr(10);
} // тук arr излиза от обхват и деструкторът се извиква автоматично
```

Така управлението на ресурса е обвързано с живота на обекта.

---

### Плитко и дълбоко копиране

Когато клас съдържа указател към динамична памет, автоматично генерираният копиращ конструктор може да извърши плитко копиране. При плитко копиране се копира само стойността на указателя, тоест адресът. Така два обекта започват да сочат към една и съща динамична памет. Това може да доведе до сериозен проблем: при унищожаване и двата обекта ще се опитат да освободят една и съща памет. Затова при класове, които управляват динамична памет, често е необходимо дълбоко копиране. При дълбоко копиране се заделя нова памет и се копира съдържанието, а не само адресът.

```cpp
class IntArray {
private:
    int* data;
    int size;

public:
    IntArray(int s)
        : data(new int[s]), size(s) {
    }

    ~IntArray() {
        delete[] data;
    }

    IntArray(const IntArray& other)
        : data(new int[other.size]), size(other.size) {
        for (int i = 0; i < size; i++) {
            data[i] = other.data[i];
        }
    }
};
```

В този пример копиращият конструктор създава отделна динамична памет за новия обект и копира стойностите от оригиналния обект.

---

### Копиращ оператор за присвояване

Копиращият оператор за присвояване се използва, когато вече съществуващ обект получава стойността на друг вече съществуващ обект.

```cpp
ClassName& operator=(const ClassName& other);
```

Разликата между копиращ конструктор и копиращ оператор за присвояване е следната:

```cpp
IntArray a(5);

IntArray b = a; // извиква се копиращ конструктор

IntArray c(10);
c = a;          // извиква се копиращ оператор за присвояване
```

Копиращият конструктор създава нов обект, а операторът за присвояване променя вече съществуващ обект. При класове, които управляват ресурси, копиращият оператор трябва да освободи стария ресурс, да задели нов и да копира съдържанието, като внимава за самоприсвояване. Както има копиращ оператор за присвояване, така има и преместващ оператор за присвояване.
---

### Rule of Zero

Ако класът не управлява ресурс директно, не трябва да се дефинират ръчно деструктор, копиращи или преместващи операции. По-добре е да се използват готови класове като `std::string`, `std::vector` или smart pointers, които сами управляват ресурсите си.

### Rule of Three

Ако класът управлява ресурс директно и е необходимо да се дефинира един от следните елементи:

- деструктор;
- копиращ конструктор;
- копиращ оператор за присвояване;

обикновено трябва да се дефинират и трите.

Причината е, че ресурсът трябва да бъде правилно освободен, копиран и присвояван.

### Rule of Five

След C++11, ако класът управлява ресурс директно, трябва да се има предвид и преместването. Затова към предишните три операции се добавят още две:

- преместващ конструктор;
- преместващ оператор за присвояване.

Така специалните функции стават пет:

- деструктор;
- копиращ конструктор;
- копиращ оператор за присвояване;
- преместващ конструктор;
- преместващ оператор за присвояване.

## Методи – декларация, предаване на параметри, връщане на резултат

Методите, наричани още член-функции, са функции, които принадлежат на даден клас. Те описват поведението на обектите от този клас и работят с неговите член-данни.

---

## Декларация и дефиниция на метод

Методите се декларират в тялото на класа. Те могат да бъдат дефинирани директно в класа или само да бъдат декларирани там, а дефиницията им да бъде написана извън класа.

### Метод, дефиниран вътре в класа

```cpp
class Point {
private:
    int x;
    int y;

public:
    void setX(int newX) {
        x = newX;
    }

    int getX() const {
        return x;
    }
};
```

---

### Метод, дефиниран извън класа

Методът може да бъде само деклариран в класа, а тялото му да бъде дефинирано извън класа чрез оператора за област на видимост `::`.

```cpp
class Point {
private:
    int x;
    int y;

public:
    void setX(int newX);
    int getX() const;
};
```

Дефиниция извън класа:

```cpp
void Point::setX(int newX) {
    x = newX;
}

int Point::getX() const {
    return x;
}
```

Записът `Point::setX` означава, че методът `setX` принадлежи на класа `Point`.

---

### Извикване на метод

Методите се извикват чрез обект от съответния клас. За достъп до публичен метод се използва операторът точка `.`.

```cpp
Point p;

p.setX(10);
std::cout << p.getX();
```

В този пример методът `setX` променя състоянието на обекта `p`, а `getX` връща стойността на член-данната `x`.

Когато работим с указател към обект, метод се извиква чрез оператора `->`.

```cpp
Point* ptr = &p;

ptr->setX(20);
std::cout << ptr->getX();
```

---

### Указателят `this`

Всеки нестатичен метод се извиква за конкретен обект. В C++ този обект е достъпен вътре в метода чрез скрития указател `this`, който сочи към обекта, за който е извикан методът. Чрез него методът достъпва член-данните на конкретната инстанция.

```cpp
class Point {
private:
    int x;

public:
    void setX(int x) {
        this->x = x;
    }
};
```

В този пример името на параметъра `x` съвпада с името на член-данната `x`. Затова чрез `this->x` ясно се посочва член-данната на обекта, а `x` без `this` означава параметъра на метода.

---

### Предаване на параметри към методи

Методите могат да приемат параметри по същия начин както обикновените функции. Параметрите служат за подаване на входни данни към метода.

 - Предаване по стойност

При предаване по стойност методът получава копие на аргумента. Промените върху параметъра не влияят на оригиналната стойност.

```cpp
void setAge(int newAge) {
    if (newAge >= 0) {
        age = newAge;
    }
}
```

 - Предаване по референция

При предаване по референция методът работи директно с оригиналния обект или променлива.

```cpp
void increase(int& value) {
    value++;
}
```

 - Предаване по константна референция

При обекти с по-голям размер често се използва предаване по константна референция. Така се избягва копиране, но методът няма право да променя подадения аргумент.

```cpp
void setName(const std::string& newName) {
    name = newName;
}
```

 - Предаване чрез указател

Метод може да приема и указател като параметър. Това позволява работа с адрес на обект или променлива.

```cpp
void setValue(int* ptr) {
    if (ptr != nullptr) {
        value = *ptr;
    }
}
```

При работа с указатели е важно да се проверява дали указателят не е `nullptr`, преди да бъде дереференциран.

---

### Връщане на резултат от метод

Методите могат да връщат резултат чрез оператора `return`. Типът на връщаната стойност се записва пред името на метода.

```cpp
int getAge() const {
    return age;
}
```

Тук методът връща стойност от тип `int`.

---

### Връщане по стойност

При връщане по стойност методът връща копие на резултата.

```cpp
int getX() const {
    return x;
}
```

Това е подходящо за прости типове като `int`, `double`, `bool` и `char`.

---

### Връщане по референция

Метод може да върне резултат по референция. Това позволява да се избегне копиране, но трябва да се използва внимателно.

```cpp
std::string& getName() {
    return name;
}
```

Този метод връща директен достъп до член-данната `name`, което позволява тя да бъде променена отвън. Това може да наруши капсулацията.

По-безопасен вариант е връщане на константна референция:

```cpp
const std::string& getName() const {
    return name;
}
```

Така се избягва копиране, но външният код не може да променя `name` директно.

---

### Константни методи

Константен метод е метод, който не променя състоянието на обекта. В C++ това се означава чрез ключовата дума `const`, поставена след списъка с параметри.

```cpp
class Person {
private:
    std::string name;
    int age;

public:
    int getAge() const {
        return age;
    }

    void setAge(int newAge) {
        age = newAge;
    }
};
```

Методът `getAge` е константен, защото само връща информация и не променя обекта. Методът `setAge` не е константен, защото променя член-данната `age`.

Константните методи могат да бъдат извиквани както за обикновени, така и за константни обекти.

```cpp
const Person p;

p.getAge(); // позволено
// p.setAge(20); // грешка, защото setAge не е const метод
```

В константен метод указателят `this` се разглежда като указател към константен обект, поради което член-данните не могат да бъдат променяни.

---

## Статични методи

Статичните методи принадлежат на класа, а не на конкретен обект. Те не получават скрит указател `this` и могат директно да достъпват само статични членове на класа.

```cpp
class Example {
public:
    static void printInfo() {
        std::cout << "Static method";
    }
};
```
---

## Наследяване
Наследяването е механизъм в обектно-ориентираното програмиране, чрез който се създават нови класове (наследници), базирани на вече съществуващи класове (базови).
Базовият (родителски) клас дефинира общи характеристики – член-данни и методи, които автоматично се наследяват от производния (наследник) клас.
Така се постига повторна употреба на код и организация на програмата в йерархия от класове.

### Производни класове
Производен клас (или наследник) е клас, който е създаден на базата на вече съществуващ клас - базов клас. Той наследява всички публични и защитени членове (променливи и методи) на базовия клас и може да добавя нови членове или да модифицира (презаписва) наследени методи.
```c++
class Base {
public:
    void foo() { std::cout << "Base foo\n"; }
};

class Derived : public Base {  // Derived е производен клас на Base
public:
    void bar() { std::cout << "Derived bar\n"; }
};
```

### Вложени класове
Вложени класове са класове, дефинирани вътре в друг клас (външен клас). Те служат за логическа групировка и капсулиране на свързана функционалност. Вложеният клас има достъп до публичните и защитените статични членове на външния клас, но за достъп до нестатичните членове се нуждае от обект на външния клас.

Пример:
```c++
class Outer {
public:
    class Inner {
    public:
        void display() { std::cout << "Inner class\n"; }
    };
};

// Достъп до вложения клас
Outer::Inner obj;
obj.display();
```

#### Видове наследяване според достъпа:
В CPP се позволява три вида наследяване, които се определят от спецификатора за достъп:
```c++
class Derived : public Base { ... };     // публично наследяване
class Derived : protected Base { ... };  // защитено наследяване
class Derived : private Base { ... };    // частно наследяване
```
- При `public` наследяване
    - `public`и `protected` членовете на базовия клас запазват достъпността си.
- При `protected`
    -  `public и `protected` членове стават `protected`.
- При `private`
    - всичко става `private`.

Пояснение: Типът наследяване по подразбиране зависи от ключовата дума, използвана за дефиниране на производния клас:
- При `class Derived : Base { ... }`; наследяването по подразбиране е `private`.
- При `struct Derived : Base { ... };` наследяването по подразбиране е `public`.

Важно е също така, че:
- Производният клас не може да достъпва директно `private` членове на базовия клас, но може да използва публични или защитени методи за достъп.
- Наследствените методи могат да бъдат презаписвани (override) ако са виртуални.

#### Пример за наследяване и разширяване на поведение
```c++
class Animal {
public:
    void eat() { std::cout << "Eating...\n"; }
};

class Dog : public Animal {
public:
    void bark() { std::cout << "Barking...\n"; }
};
// Класът Dog наследява метода eat() от Animal и добавя собствен метод bark().
```

### (Допълнително*)Наследяване ("is-a") срещу композиция и агрегацията("has-a")
Композицията представлява „силна“ връзка между обектите: ако обект A съдържа обект B чрез композиция, то B не може да съществува независимо от A.
```c++
class Engine {
    // ...
};

class Car {
private:
    Engine engine; // композиция
};
// Тук engine е част от Car, създава се и се унищожава заедно с него.
```

Агрегацията е „по-слаба“ връзка, при която обектът B може да съществува независимо от обекта A. Обикновено се реализира чрез указатели или референции.
```c++
class Engine {
    // ...
};

class Car {
private:
    Engine* engine; // агрегация
public:
    Car(Engine* e) : engine(e) {}
};
// Тук Car използва Engine, но не го притежава — двигателят може да се използва и от други обекти.
```

В обобщение, макар и наследяването, композицията и агрегацията да служат за преизползване на код, изборът между тях зависи от логическата връзка между обектите. Наследяването (връзка "is-a") е подходящо, когато даден клас е специализация на друг и се цели използване на полиморфизъм. От друга страна, композицията и агрегацията (връзка "has-a") се използват за изграждане на сложни обекти чрез включване на други като компоненти, като това често води до по-гъвкав дизайн и по-слаба степен на свързаност между класовете.

## Статични полета и методи
Статичните членове (данни и методи) са асоциирани със самия клас, а не с конкретна инстанция (обект) на класа. Това означава, че съществува само едно-единствено копие на статичните данни, което се споделя между всички обекти от този клас.
- Статични данни (Static Data Members):
    - Декларират се с ключовата дума `static` вътре в класа, но задължително се дефинират и инициализират извън него, в глобалното пространство.
    - Пазят се в статичната/глобалната памет
        - съществуват през цялото време на изпълнение на програмата.
- Статични методи (Static Member Functions):
    - Не получават скрит this указател, тъй като не са свързани с конкретен обект.
    - Поради това могат да достъпват директно само други статични членове (данни или методи) на класа.
    - Извикват се чрез името на класа и оператора за обхват (`::`), например `ClassName::staticMethod()`.
```c++
#include <iostream>

class Widget {
public:
    Widget() { 
        count++; // Увеличаваме общия брояч при създаване на всеки обект
    }
    ~Widget() { 
        count--; // Намаляваме общия брояч при унищожаване на всеки обект
    }

    // Статичен метод за достъп до статичния брояч
    static int getCount() { 
        return count; 
    }

private:
    // Декларация на статичната член-данна
    static int count; 
};

// Дефиниция и инициализация на статичната член-данна
int Widget::count = 0; 

int main() {
    std::cout << "Начален брой: " << Widget::getCount() << std::endl; // 0

    Widget w1;
    Widget w2;
    std::cout << "Брой след създаване на w1 и w2: " << Widget::getCount() << std::endl; // 2

    {
        Widget w3;
        std::cout << "Брой след създаване на w3: " << w1.getCount() << std::endl; // 3 (достъп и през обект)
    } // w3 се унищожава тук

    std::cout << "Брой след унищожаване на w3: " << Widget::getCount() << std::endl; // 2
    return 0;
}
```

### Статични членове и наследяване
Поведението на статичните членове при наследяване е ключово за разбиране: те не се дублират в класа-наследник.
- Статични данни    
    - Съществува само едно копие на статичната променлива, дефинирана в базовия клас.
    - Всички инстанции на базовия клас и всички инстанции на неговите наследници споделят и променят същата тази променлива.
- Статични методи
    - Статичните методи на базовия клас могат да бъдат извикани и през името на класа-наследник.
    - Важно е да се отбележи, че те не могат да бъдат виртуални (не участват в полиморфизма).
    - Ако наследник дефинира статичен метод със същото име, той скрива (hides) метода на базовия клас, а не го предефинира (overrides).
```c++
#include <iostream>

class Entity {
public:
    // Брояч, общ за всички същества в играта
    static int entity_count;

    Entity() {
        entity_count++;
    }

    static int getEntityCount() {
        return entity_count;
    }
};

// Дефиниция на статичния член от базовия клас
int Entity::entity_count = 0;

// Player наследява Entity
class Player : public Entity {
public:
    std::string name;

    Player(std::string name) : name(name) {
        // Конструкторът на базовия клас (Entity) се извиква автоматично,
        // увеличавайки същия брояч entity_count.
        std::cout << "Създаден е играч: " << name << std::endl;
    }
};

int main() {
    std::cout << "Общо същества в началото: " << Entity::getEntityCount() << std::endl; // 0

    Player p1("Gandalf");
    Player p2("Aragorn");
    Entity monster;

    // Достъпваме ЕДНА И СЪЩА променлива през различните класове
    std::cout << "Общо същества (през Entity): " << Entity::getEntityCount() << std::endl; // 3
    std::cout << "Общо същества (през Player): " << Player::getEntityCount() << std::endl; // 3
    std::cout << "Общо същества (директен достъп през Player): " << Player::entity_count << std::endl; // 3
}
```
